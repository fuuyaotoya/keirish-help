# 電帳法について — 保存要件とファイル名規則

ケイリッシュは **電子帳簿保存法 (電帳法) スキャナ保存・電子取引保存** の要件を満たすよう設計されています。

## 電帳法が定める 4 つの要件

| 要件 | 内容 | ケイリッシュの対応 |
|---|---|---|
| 真実性 | 改ざん防止 | SHA-256 ハッシュ + タイムスタンプ |
| 可視性 | 検索性確保 | 索引 CSV を自動同梱 |
| 可読性 | 200dpi / 24bit カラー | スキャン解像度を自動チェック |
| 検索性 | 日付・金額・取引先で検索可能 | ファイル名 + 索引 CSV |

## ファイル名規則

アップロードした領収書は、以下のテンプレートで自動リネームされます。

```
YYYYMMDD_金額_取引先.拡張子
```

例: `20260515_5500_スターバックス.pdf`

> **順序**: 国税庁は項目の順序を規定していません。ケイリッシュは **日付→金額→取引先** の順を採用しています（日付でのディレクトリソート時に金額が続くため視認性が高い）。

### OCR 誤読の修正

スキャン後、ファイル名をダウンロード前に直接編集できます。AI が金額を誤読した場合（例: 1000 → 100）、画面上のテキストボックスで修正してからダウンロードしてください。

### 外貨建の場合

円換算額が確定している場合:

```
YYYYMMDD_円換算額JPY_通貨コード外貨額.拡張子
```

例: `20260515_15000JPY_USD100.00_AmazonWebServices.pdf`

円換算額が未確定の場合（TTM レート未入力）:

```
YYYYMMDD_通貨コード外貨額_取引先.拡張子
```

例: `20260515_USD100.00_AmazonWebServices.pdf`

対応通貨: USD / EUR / CNY / KRW / GBP / AUD / HKD / SGD / CAD / CHF（主要10通貨）

#### TTM レートについて

外貨建領収書のレート換算は、初回スキャン時に換算方法を選択できます:

| 方式 | 説明 | 適合シーン |
|------|------|-----------|
| A: 日次参照レート | 市場参照レート（ExchangeRate-API / ECBデータ）を自動取得 | 毎日少量の外貨決済 |
| C: 月次固定TTM | 毎月1日の銀行公示TTMを月中一律適用 | 大量の外貨取引がある法人 |

> **重要**: 自動取得レートは市場参照レートであり、銀行公示TTM（電信売買相場の仲値）ではありません。税務上の正確性が必要な場合は、銀行の公示TTMを確認して手動で入力してください。法人税法は換算方法の **継続適用** を要件とするため、どちらかを選択して一貫して使用することで要件を満たします。

## 索引 CSV (検索性確保)

エクスポート時にダウンロードされる ZIP に「索引 CSV」が同梱されます。
このファイルには以下の列が含まれ、税務調査時の検索性を満たします。

| 列 | 説明 |
|---|---|
| 取引日 | YYYY-MM-DD |
| 金額 | 円 |
| 取引先 | 正規化済み |
| 勘定科目 | 確定後 |
| 電帳法ファイル名 | リネーム後 |
| 元ファイル名 | アップロード時 |

## 保存期間

- 法人: 7 年 (繰越欠損金がある場合は 10 年)
- 個人事業主: 7 年

ケイリッシュは Phase 2 (ALL プラン) で、退会後も保存期間が満了するまでデータを保持する設計です。

## スキャン品質と圧縮率

### 解像度

- **最低基準**: 200dpi（A4 = 1654×2339px = 3.87MP）
- **ケイリッシュ スキャンページ**: 長辺 3000px に制限（A4縦 = 約257dpi）
- **ケイリッシュ 経理保存ページ**: フル解像度保持（10MB上限）

スキャンページは AI OCR 用途（エフェメラル）のため 3000px で十分。電帳法保存目的の `/accounting` ルートは原寸を保持します。

### JIIMA 圧縮率基準（1/20 以上）

電帳法スキャナ保存の JIIMA 認定基準は、**圧縮後サイズ ≥ 非圧縮 RAW ビットマップの 1/20**（圧縮率 20:1 以下）。

| A4 解像度 | 24bit RAW | 1/20 = 最小保存サイズ | ケイリッシュ (quality 0.90) |
|-----------|-----------|----------------------|--------------------------|
| 200dpi (3.87MP) | 11.6MB | 580KB | ≈ 800KB-1.5MB ✅ |
| 257dpi (3000px) | 19.2MB | 960KB | ≈ 1.2MB-2.5MB ✅ |

JPEG quality 0.90 は実測で 10:1 前後の圧縮率となり、JIIMA の 20:1 基準を安全にクリアします。
